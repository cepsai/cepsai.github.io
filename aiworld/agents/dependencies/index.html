Methodology

Sources. "Depends on" rows come from the latest published version's manifest as indexed by Ecosyste.ms (PyPI install_requires / npm dependencies). Optional/extras dependencies are grouped under their feature key (e.g. extra == "anthropic"). "Who depends on it" rows come from Ecosyste.ms's dependent_packages endpoint, sorted by last-month downloads. Counts in the summary table compare libraries.io's own crawl against Ecosyste.ms's.

Why the two counts disagree. libraries.io has been crawling registries since 2017 and counts any historical dependency relationship between packages — including dead, abandoned, and now-broken packages. Ecosyste.ms re-indexes more recent state and tends to under-count older relationships but over-counts where libraries.io has gone stale (it has been semi-maintained since the original maintainer left Tidelift). For packages where Ecosyste.ms reports 0 dependent packages (smolagents, pydantic-ai, @mastra/core, browser-use, fast-agent, autogen-agentchat) the package is too new or its dependents are not yet in Ecosyste.ms's reverse-dep index — libraries.io's number is more accurate for those.

Caveats. "Dependent repositories" counts public GitHub repos that import the package via a manifest — both services miss vendored installs, monorepo internal packages, and anything pinned outside a recognised manifest. Download counts are last-month from PyPI/npm via Ecosyste.ms. Star/fork numbers reflect the linked GitHub repo at last sync.
